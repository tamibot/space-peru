# Backlog

Tareas pendientes, ideas, roadmap. Formato simple: un archivo por iniciativa o un solo `tasks.md` para tickets sueltos.

## Archivos
- **`plan-de-trabajo.md`** — plan maestro de 6 fases (Fase 1 → Fase 6). **Documento canónico de cómo ejecutamos.**
- `roadmap.md` — visión por fases (MVP → v1 → v2). Resumen del plan-de-trabajo.
- `tasks.md` — tickets sueltos sin asignar
- `ideas.md` — ideas crudas sin priorizar
- `decisiones.md` — log de decisiones tomadas (ADR ligero)

## Convención de tickets

```
- [ ] [PRIO] [AREA] Descripción corta — owner — ETA
```

`PRIO`: P0 / P1 / P2  
`AREA`: research / landing / app / social / infra / branding / legal
